PC環境を極力汚さず（システム環境変数やグローバル領域をいじらず）に、プロジェクト単位で完結させる「ポータブル＆クリーン」な環境構築手順をご案内します。


## 1. プロジェクトフォルダの作成と仮想環境（venv）の構築

Pythonには、プロジェクトごとにパッケージ（PySide6など）を隔離してインストールできる「仮想環境（venv）」という標準機能があります。これを使えば、PC全体のPython環境を汚すことはありません。

1. コマンドプロンプト、またはPowerShellを開きます。
    
2. 開発用のフォルダを作成し、その中に移動します。
    
    PowerShell
    
    ```
    mkdir Myffmpeg
    cd Myffmpeg
    ```
    
3. 仮想環境（`.venv` という名前の隠しフォルダ）を作成します。
    
    PowerShell
    
    ```
    python -m venv .venv
    ```
    
4. 仮想環境を有効化（アクティベート）します。
    
    PowerShell
    
    ```
    .\.venv\Scripts\activate
    ```
    
    _※成功すると、コマンドラインの先頭に `(.venv)` と表示されます。以降の作業はこの状態で行ってください。_
    
    _※PowerShellで「スクリプトの実行が無効になっている」というエラーが出た場合は、一度だけ以下のコマンドを実行して権限を付与してから、再度アクティベートしてください。_
    
    `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`
    

## 2. PySide6のインストール

仮想環境が有効化されている `(.venv)` 状態で、以下のコマンドを実行します。

これにより、PC全体ではなくこのプロジェクトの `.venv` フォルダの中だけにPySide6がインストールされます。

PowerShell

```
pip install PySide6
```

## 3. FFmpegのポータブル導入（PATHを汚さない方法）

インストーラーを使わず、実行ファイル（`.exe`）だけをダウンロードしてプロジェクト内に配置します。これにより環境変数（PATH）を一切汚さずに済みます。

1. **ダウンロード:**
    
    FFmpegのWindows向け公式推奨ビルド配布サイト（[gyan.dev](https://www.gyan.dev/ffmpeg/builds/)）にアクセスします。
    
    - `ffmpeg-release-essentials.zip` をダウンロードしてください。
        
2. **展開と配置:**
    
    - ダウンロードしたzipファイルを解凍します。
        
    - 解凍したフォルダの `bin` フォルダの中にある **`ffmpeg.exe`** と **`ffprobe.exe`** の2つのファイルをコピーします。
        
3. **プロジェクトに組み込む:**
    
    - 先ほど作成した `Myffmpeg` フォルダの中に `tools` という新しいフォルダを作ります。
        
    - その `tools` フォルダの中に、コピーした2つの `.exe` ファイルを貼り付けます。
        

## 4. 最終的なフォルダ構成の確認

ここまでの作業で、フォルダ構成が以下のようになっていれば環境構築は完璧です。

Plaintext

```
Myffmpeg/
 │
 ├─ .venv/          ← 仮想環境（PySide6などがこの中に入っています）
 │
 ├─ tools/          ← FFmpeg置き場（PC環境を汚しません）
 │   ├─ ffmpeg.exe
 │   └─ ffprobe.exe
 │
 └─ main.py         ← 今後ここにAIが出力したコードをコピペして作っていきます（新規作成してください）
```

**▶ 今後の開発の進め方**

作業を再開するときは、必ず毎回プロジェクトフォルダ（`Myffmpeg`）をターミナルで開き、`.\.venv\Scripts\activate` で仮想環境を有効にしてから `python main.py` でアプリを起動するようにしてください。

この状態になれば、先ほどの「仕様・実装計画書」をAIに渡して、`main.py` 用のコードを生成させる準備は完了です。
