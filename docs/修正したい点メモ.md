## 重大な問題
### 2026/7/24 ハードウェアエンコーダは2パスエンコードができない、モノによってpresetの値が違うのでエラー出る

#### サイズ指定（2パスエンコード）について
CPU→従来通り
NVIDIA→`-rc vbr`（可変ビットレート）と `-multipass 2pass_full`を組み合わせて1回で処理します。
```bash
ffmpeg -i input.mp4 -c:v h264_nvenc -rc vbr -b:v 1237k -maxrate 1800k -bufsize 2500k -multipass 2pass_full -c:a aac -b:a 128k output.mp4
```
IntelGPU →`-look_ahead 1`

```bash
ffmpeg -i input.mp4 -c:v h264_qsv -b:v 1237k -maxrate 1800k -look_ahead 1 -look_ahead_depth 40 -c:a aac -b:a 128k output.mp4
```
AMD GPU→レートコントロールに `-rc vbr_peak` を指定します。
```bash
ffmpeg -i input.mp4 -c:v h264_amf -rc vbr_peak -b:v 1237k -maxrate 1800k -c:a aac -b:a 128k output.mp4
```

def get_quality_args(encoder: str, quality_val: int) -> List[str]:
    """エンコーダーの特性に合わせて、正しい品質指定引数リストを生成するヘルパー関数"""
    val_str = str(quality_val)

    # 1. AMD (AMF): -rc cqp とセットで -qp_i, -qp_p, -qp_b を一括指定
    if "amf" in encoder:
        return [
            "-rc",
            "cqp",
            "-qp_i",
            val_str,
            "-qp_p",
            val_str,
            "-qp_b",
            val_str,
        ]

    # 2. NVIDIA (NVENC): -rc vbr モードと -cq をセットで指定
    elif "nvenc" in encoder:
        return ["-rc", "vbr", "-cq", val_str]

    # 3. Intel (QSV): -global_quality を指定
    elif "qsv" in encoder:
        return ["-global_quality", val_str]

    # 4. CPU (libx264, libx265, libsvtav1): -crf を指定
    else:
        return ["-crf", val_str]
