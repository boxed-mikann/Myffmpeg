# 🚀 Myffmpeg 仕様・実装計画書

## 1. プロジェクト概要

- **アプリ名:** Myffmpeg
    
- **目的:** よく使うFFmpegコマンド（動画圧縮、2パスビットレート計算、一括処理）の自動化とGUI化。
    
- **ターゲット:** 自分用（個人開発・技術学習を兼ねる）。
    
- **提供価値:** 面倒なビットレート計算の自動化、一括バッチ処理、GUI上での10秒プレビュー、プリセットによる設定管理。
    

## 2. 技術選定・動作環境

- **言語:** Python 3.x
    
- **GUIフレームワーク:** PySide6
    
- **バックエンド:** FFmpeg, ffprobe（実行ファイル同梱）
    
- **エンコーダ:** CPU（libx264/libx265）、およびIntel QSV（h264_qsv/hevc_qsv）を対象とする。
    
- **パス解決:** 将来的なPyInstallerでのビルドを見据え、`sys._MEIPASS` を考慮した絶対パスで `ffmpeg` および `ffprobe` を呼び出す。
    

## 3. UI画面構成（3ブロック構成）

画面は大きく以下の3つのペイン（またはタブ・スプリッター）で構成する。

1. **入力ブロック（Input）**
    
    - ファイル・フォルダのドラッグ＆ドロップ、またはダイアログ選択。複数追加可能
        
    - 拡張子フィルタリング（動画ファイルのみ許可）。
        
    - **リスト表示:** ファイル名、元解像度、元サイズ、処理対象チェックボックス。総ファイル数と総サイズを上部に表示。
        
    - **データ構造:** 各リストアイテムは、`ffprobe` で取得したJSONデータをそのまま辞書型（dict）として保持する。
        
2. **設定ブロック（Settings）**
    
    - 目的別にタブで切り替え可能（詳細は後述）。
        
    - 自由記述のオプション追加用テキストボックス。
        
    - プリセットの保存・読み込み（`settings.json` へ保存）。
        
    - プレビュー対象ファイルのドロップダウン選択 ＋ プレビュー実行ボタン。
        
    - PySide6 `QMediaPlayer` ＋ `QVideoWidget` を用いたプレビュー再生領域。
        
3. **出力ブロック（Output）**
    
    - 処理ログ、実行される最終コマンドラインの表示。
        
    - 生成されたファイルサイズ、成功数、失敗数のステータス表示。
        
    - 「実行（ジョブ開始）」および「キャンセル」ボタン。
        
    - 全体の進捗を示すプログレスバー。
        

## 4. 設定ブロックの詳細（タブ別仕様）

### タブA: 映像消去

- **目的:** 動画から映像ストリームを削除し、音声のみを抽出・または音声のみのファイルにする。
    
- **処理:** FFmpegオプション `-vn` `-f mp3` （映像なし）を使用。
    
- **出力ファイル名:** `[元ファイル名]_novideo.[拡張子]` など。実行時に処理フラグリストに登録し、次回以降の重複処理を防ぐ。
    

### タブB: 圧縮（品質ベース）

- **目的:** 画質を保ったままサイズを抑える（1パス）。
    
- **設定項目:** 解像度、フレームレート、エンコーダ（CPU / QSV）、画質（CRF等のエンコーダ依存値）、処理速度（preset）、オーディオモノラル化、オーディオビットレート。
    

### タブC: サイズ指定圧縮（2パス処理）

- **目的:** 指定された目標ファイルサイズ（MB）に収まるよう、ビットレートを自動計算して2パスエンコードを行う。
    
- **設定項目:** 解像度、フレームレート、エンコーダ（QSV対応）、オーディオモノラル化、オーディオビットレート、**目標サイズ（MB）**。
    
- **事前計算ロジック（Python側で実行）:**
    
    1. `音声サイズ(MB) = (音声ビットレート(kbps) × 動画秒数) / 8000`
        
    2. `映像割当サイズ(MB) = 目標サイズ(MB) - 音声サイズ(MB)`
        
    3. `映像ビットレート(kbps) = (映像割当サイズ(MB) × 8000) / 動画秒数`
        
    4. ※`映像割当サイズ` が0以下、または極端に低い場合はエラーダイアログを出し、実行をブロックする。
        

## 5. バックエンド処理・プロセス制御（超重要要件）

- **QThreadの採用:**
    
    - UIのフリーズを防ぐため、FFmpegの実行プロセス（`subprocess.Popen`）はすべて `QThread` 内で行う。
        
- **進捗（プログレス）の取得:**
    
    - FFmpegの引数に `-progress -` を渡し、標準出力に流れる機械向けログを取得。
        
    - `out_time_us=XXXXXX`（マイクロ秒）の行をパースし、`ffprobe` で取得済みの「動画総秒数」と照らし合わせて進捗率（%）を算出し、UIへシグナル送信する。
    - ffmpegの元の実行の様子も見られるようにする。 
        
- **キャンセルの実装:**
    
    - キャンセルボタン押下時、QThread内の `subprocess` に対して `terminate()` を発行。
        
    - `wait()` でプロセスの完全終了を待機。
        
    - 処理途中で生成された不完全な出力ファイルを `os.remove()` で削除する。
        
- **上書き確認の処理:**
    
    - コマンドライン上の `y/N` 入力は使用しない。
        
    - 実行開始前、Python側で `os.path.exists()` を用いて出力先ファイルの存在確認を行う。
        
    - 存在する場合は PySide6 の確認ダイアログを出し、ユーザーが「はい」を選んだ場合のみ FFmpeg に `-y` オプションを付与して実行する。
        

## 6. プレビュー機能の実装

- **対象:** 入力リストから選択された任意の動画。
    
- **動作:** 現在の設定（タブ）の内容を元に、`-t 10`（開始から10秒間、あるいは指定位置から10秒間）のプレビュー用エンコードを一時フォルダに実行。最終サイズ指定での実行の場合は、プレビューの処理はサイズ指定を最終指定サイズ\*10/動画時間とすること。
    
- **再生:** エンコード完了後、PySide6の `QMediaPlayer` で再生を開始する。
